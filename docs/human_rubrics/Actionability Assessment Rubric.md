## STEP 9: Final Scoring

### Actionability Score (0-5)

**Decision Tree:**

```
START → Count recommendations at Level 3-4 specificity

≥3 specific recommendations (Level 3-4)?
├─ YES → Check implementation guidance
│   └─ Most have HOW details?
│       ├─ YES → Check context
│       │   └─ Context clear for most?
│       │       ├─ YES → Check success criteria
│       │       │   └─ Metrics specified?
│       │       │       ├─ YES → Check resources
│       │       │       │   └─ Resources disclosed?
│       │       │       │       ├─ YES → Check prioritization
│       │       │       │       │   └─ Priority/sequence clear?
│       │       │       │       │       ├─ YES → Check examples
│       │       │       │       │       │   └─ Examples provided?
│       │       │       │       │       │       ├─ YES → SCORE = 5
│       │       │       │       │       │       └─ NO → SCORE = 4
│       │       │       │       │       └─ NO → SCORE = 4
│       │       │       │       └─ NO → SCORE = 4
│       │       │       └─ NO → SCORE = 3-4
│       │       └─ NO → SCORE = 3-4
│       └─ NO → SCORE = 3
│
2-3 specific OR several moderate (Level 2-3)?
├─ YES → Some guidance + some context → SCORE = 4
└─ NO → SCORE = 3

1-2 specific OR several moderate?
└─ Minimal guidance → SCORE = 3

Recommendations vague (mostly Level 1-2)?
└─ Little guidance → SCORE = 2

Very vague (Level 1)?
└─ SCORE = 1

No recommendations?
└─ SCORE = 0
```

### Detailed Scoring Rubric

**SCORE 5: Exemplary** ✓ All criteria met:
- [ ] ≥3 specific recommendations (Level 3-4)
- [ ] Implementation guidance provided
- [ ] Clear contextual applicability
- [ ] Success criteria & metrics specified
- [ ] Resource requirements disclosed
- [ ] Recommendations prioritized/sequenced
- [ ] Practical examples/templates included

---

**SCORE 4: Strong** ✓ Most criteria met:
- [ ] 2-3 specific recommendations (Level 3-4)
- [ ] Some implementation guidance
- [ ] Context mostly clear
- [ ] Some success criteria
- [ ] Some resource info
- [ ] Implicit or partial prioritization
- [ ] Some examples

**What's missing:** Less detailed steps, fewer metrics, limited resources, weaker prioritization

---

**SCORE 3: Adequate** ✓ Minimum met:
- [ ] 1-2 specific OR several moderate (Level 2-3)
- [ ] Minimal implementation guidance
- [ ] Some context
- [ ] Vague success criteria
- [ ] Limited resource info
- [ ] No clear prioritization
- [ ] Minimal examples

**What's missing:** Many lack specificity, sparse implementation details, incomplete context

---

**SCORE 2: Weak** ⚠️ Below minimum:
- [ ] Vague recommendations (Level 1-2)
- [ ] Little/no implementation guidance
- [ ] Context minimal/absent
- [ ] No success criteria
- [ ] No resource discussion
- [ ] No prioritization
- [ ] No examples

**Key problem:** Aspirational advice without practical details

---

**SCORE 1: Poor** ⚠️ Critically deficient:
- [ ] Very vague suggestions (Level 1)
- [ ] No guidance on implementation
- [ ] Cannot act on advice

---

**SCORE 0: No Actionability**
- [ ] No recommendations provided

## Troubleshooting

### Issue: "Is this specific enough for Level 3 or 4?"

**Solution:**
- **Level 4**: Numbers/quantities ("5-7 minutes", "top-k=5")
- **Level 3**: Clear action with detail ("segment with checkpoints")
- If it has a number → likely Level 4
- If you know exactly what to do → Level 3
- If you need to figure out details → Level 2

---

### Issue: "How much implementation guidance is 'enough'?"

**Solution:**
**Complete (Quality 3)** = You could start implementing right now
**Partial (Quality 2)** = You have general direction but need to research specifics
**Minimal (Quality 1)** = You know WHAT but not HOW
